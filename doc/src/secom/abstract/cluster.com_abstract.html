Cluster secom.abstract

secom has four abstract classes for communicating over a serial device:


Device
Represents a serial device. Used for reading from and writing to a serial
device. Both Posix and Windows have a concrete version called COM_DEVICE.
Settings
Device communication properties such as baud rate and parity
Timeouts
Device properties that define if and when a read will time out
Constants
Use these constants to alter device properties. Both Posix and Windows have
a concrete version called COM_CONTROL_CONSTANTS
